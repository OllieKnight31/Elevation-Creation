## GHL Inbound Webhook URLs (Generated Inside GHL)

These webhook URLs are auto-generated by GHL when you create an "Inbound Webhook" trigger on a workflow. You must copy the generated URL from GHL and paste it into the corresponding external service's webhook settings.

| GHL Workflow | Trigger Type | Where to Paste the Generated URL | Purpose |
|-------------|-------------|----------------------------------|---------|
| **WF1 -- Registration Trigger** | Inbound Webhook (if using external reg page, not GHL form) | Paste into external registration page form action / webhook config | Receives new registrations and creates contacts + opportunities in GHL |
| **WF3 -- Webinar Attendance Handler** | Inbound Webhook | Paste into **WebinarJam** webhook settings for "Attendee Joined" event | Marks contacts as attended, removes from reminder sequence, enrols in post-webinar sequence |
| **WF6 -- Call Booked + Reminders** | Inbound Webhook | Paste into **Calendly** webhook settings for `invitee.created` event | Marks contacts as call-booked, triggers call reminder sequence |

**Format:** GHL generates URLs like:
```
https://services.leadconnectorhq.com/hooks/[unique-id]
```
These are unique per workflow trigger and are only available after creating the trigger step in GHL.

## External Service Webhook Configuration

### WebinarJam
1. Go to WebinarJam > Webinar Settings > Integrations > Webhooks
2. Add webhook for **Registration**: point to `https://webinar-system-five.vercel.app/api?source=webinarjam`
3. Add webhook for **Attendee Joined**: point to the GHL-generated URL from **WF3** trigger
4. (Optional) Add webhook for **Attendee Left** / **Replay Watched** if tracking watch time

### Calendly
1. Go to Calendly > Developer Settings > Webhooks (or use API to create subscription)
2. Subscribe to `invitee.created` event
3. Point to **both**:
   - `https://webinar-system-five.vercel.app/api?source=calendly` (for tracker sheet updates)
   - The GHL-generated URL from **WF6** trigger (for GHL automation)
4. The Calendly PAT is already configured in .env.local

### Meta Ads
1. Meta data is pulled via the sync endpoint (`?action=sync_meta`), not pushed via webhook
2. The POST endpoint (`?source=meta`) exists for manual/automated data pushes if needed
3. Meta credentials still PENDING in .env.local: `META_ACCESS_TOKEN`
